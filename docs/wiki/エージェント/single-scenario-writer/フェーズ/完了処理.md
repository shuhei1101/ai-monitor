# 完了処理

シナリオ合意（`議論中` 除去）を受けて、PR の `確認:single-scenario-writer` を除去し、親 story Issue に確認ラベルを渡して役割を終える。

## 手順

### 自分宛コメントの選別

未回答・未対応の自分宛コメントが残る場合は「応答ループ」に戻ってユーザーに確認質問を投げる（以降の手順は実行しない）。

### 一括 Resolve

MCP `resolve_comments` を呼ぶ:
- `node_ids`: 自分宛コメントの `node_id` 配列

### ラベル除去

MCP `transition_phase` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true
- `remove_labels_`:
  - `$AI_MONITOR_LABEL_CONFIRM_SINGLE_SCENARIO_WRITER` の値
- `add_labels_`: なし

### 確認ラベルの引き継ぎ

$pr_number の PR 本文の `## 紐づく Issue` から親 story Issue 番号を得る。

MCP `add_labels` を呼ぶ:
- `number`: 親 story Issue 番号
- `is_pr`: false
- `labels`:
  - `$AI_MONITOR_LABEL_CONFIRM_STORY_CONDUCTOR` の値

### 完了報告の投稿

MCP `comment` を呼ぶ:
- `number`: 親 story Issue 番号
- `is_pr`: false
- `sender`: `single-scenario-writer`
- `receiver`: `story-conductor`
- `body`: 単一シナリオ設計の完了報告（シナリオファイルへのリンク + 正常 / 異常シナリオの要点 + 確認後の Resolve 依頼）

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `single-scenario-writer`
- `number`: $pr_number
