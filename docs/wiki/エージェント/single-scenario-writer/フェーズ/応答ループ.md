# 応答ループ

シナリオ作成後の待機中に来たユーザーフィードバックに応じてシナリオ md を修正 push し、再び待機に入る。

## 手順

### 内容に応じた対応

| フィードバックの種別 | 対応 |
| --- | --- |
| フロー図のステップ追加・削除・順序変更 | シナリオ md の sequenceDiagram を修正 |
| セットアップ / 期待値の追加・修正 | 該当セクションを修正 |
| 異常シナリオの追加 | 該当シナリオファイルに `## 異常シナリオ（{条件}）` を追加 |
| 質問への回答 | 回答を該当コメントに返信追記（本文・シナリオファイルは触らない） |

シナリオ修正の場合は story ブランチの worktree で:

- `git add docs/wiki/設計図/シナリオ/単一ユースケース/{UC名}.md`
- `git commit -m "{UC名}単一シナリオを{修正内容の要約}"`
- `git push`

修正内容は該当コメントに MCP `reply_comment` で返信追記する:
- `comment_node_id`: 返信対象コメントの node_id
- `sender`: `single-scenario-writer`
- `receiver`: ユーザーログイン名
- `body`: 修正内容の要約

### assignee 再設定

MCP `set_assignee` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `single-scenario-writer`
- `number`: $pr_number
