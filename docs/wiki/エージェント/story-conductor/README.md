# story-conductor

## フェーズ一覧

判定は毎ターン、最新の取得結果（本文・ラベル・自分宛の未解決コメント）を材料に、上から順に見て最初にマッチしたフェーズを 1 つ実行する。

| フェーズ | 起動条件 |
| --- | --- |
| [初期処理](./フェーズ/初期処理.md) | 毎ターン最初に必ず実行（以降の判定材料を取得） |
| [要件確定（初回）](./フェーズ/要件確定（初回）.md) | 本文の 4 セクション未記入（未作業） |
| [応答ループ](./フェーズ/応答ループ.md) | `議論中` あり + 未解決の自分宛コメントがある |
| [要件確定（完了処理）](./フェーズ/要件確定（完了処理）.md) | `議論中` なし + 本文確定済み + story Draft PR 未作成 |
| [子subsystem起票（初回）](./フェーズ/子subsystem起票（初回）.md) | 未解決の自分宛コメントに single-scenario-writer の完了報告があり、story 本文の依存順が未記入 |
| [子subsystem起票（逐次）](./フェーズ/子subsystem起票（逐次）.md) | 未解決の自分宛コメントに subsystem-conductor のインターフェース確定報告があり、story 本文の依存順に `未起票` が残る |
| [統合テスト起動](./フェーズ/統合テスト起動.md) | 未解決の自分宛コメントに subsystem-conductor の完了報告がある |
| [バグ差し戻し（方針確認）](./フェーズ/バグ差し戻し（方針確認）.md) | 未解決の自分宛コメントに single-scenario-writer の失敗報告があり、対応方針案コメント未投稿 |
| [バグ差し戻し（実行）](./フェーズ/バグ差し戻し（実行）.md) | `議論中` なし + 対応方針案コメント投稿済み（未 Resolve） |
| [バグ差し戻しの中継（下り）](./フェーズ/バグ差し戻しの中継（下り）.md) | 未解決の自分宛コメントに epic-conductor のバグ差し戻しがある |
| [バグ差し戻しの中継（上り）](./フェーズ/バグ差し戻しの中継（上り）.md) | 未解決の自分宛コメントに subsystem-conductor の修正完了報告があり、story Issue が open（バグ差し戻し由来で reopen 済み） |
| [storyマージ](./フェーズ/storyマージ.md) | 未解決の自分宛コメントに single-scenario-writer の全 pass 報告がある |
