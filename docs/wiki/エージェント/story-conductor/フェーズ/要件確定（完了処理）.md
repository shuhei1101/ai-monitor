# 要件確定（完了処理）

承認された要件をもとに story Draft PR を作成し、single-scenario-writer に引き継ぐ。

## 手順

### 自分宛コメントの選別

未回答・未対応の自分宛コメントが残る場合は「応答ループ」に戻ってユーザーに確認質問を投げる（以降の手順は実行しない）。

### 一括 Resolve

MCP `resolve_comments` を呼ぶ:
- `node_ids`: 自分宛コメントの `node_id` 配列

### Draft PR の作成

親 epic の type ラベル（要件確定で判定済み・`type:feat` / `type:bug` 等）+ 親 epic ブランチ名の `{type}/epic/{ドメイン}` から取り出したドメイン部 + 要件確定（初回）で特定した UC 名から story ブランチを組み立てる。

MCP `worktree_create` を呼ぶ:
- `branch`: `{type}/story/{ドメイン}/{UC名}`

続けて MCP `create_draft_pr` を呼ぶ:
- `head_branch`: 作成したブランチ
- `base_branch`: 親 epic ブランチ
- `title`: story のタイトル
- `body`: `## 紐づく Issue` のみ（`- #$issue_number`）

### 監視面の登録

MCP `add_watch_targets` を呼ぶ:
- `agent_name`: `story-conductor`
- `number`: $issue_number
- `watch_numbers`: 作成した PR の番号

### 次フェーズの割り当て

MCP `add_labels` を呼ぶ:
- `number`: 作成した PR の番号
- `is_pr`: true
- `labels`:
  - `$AI_MONITOR_LABEL_CONFIRM_SINGLE_SCENARIO_WRITER` の値

### ラベル除去

MCP `transition_phase` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false
- `remove_labels_`:
  - `$AI_MONITOR_LABEL_CONFIRM_STORY_CONDUCTOR` の値
- `add_labels_`: なし

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `story-conductor`
- `number`: $issue_number
