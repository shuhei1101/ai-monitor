# 統合テスト起動

subsystem-conductor の完了報告を受けて、全子 subsystem の close 状態に応じて統合テストを single-scenario-writer に委任する（未完了があれば状況確認）。

## 手順

### 全子 subsystem の確認

初期処理で取得した Sub-issue の state を確認する。

### 完了報告の Resolve と分岐

MCP `resolve_comments` で subsystem-conductor の完了報告コメントを Resolve する。

- 全子 subsystem が closed の場合、story PR に MCP `add_labels` を呼ぶ:
  - `number`: story PR の番号（初期処理の監視面から取得）
  - `is_pr`: true
  - `labels`:
    - `$AI_MONITOR_LABEL_CONFIRM_SINGLE_SCENARIO_WRITER` の値（統合テストの委任。初回実装か再テストかの判定は writer が行う）
- open の subsystem が残っている場合、open の各 subsystem Issue に状況確認を送る:
  1. MCP `comment` を呼ぶ（`number`: open の subsystem Issue 番号・`is_pr`: false・`sender`: `story-conductor`・`receiver`: `subsystem-conductor`・`body`: 進捗とブロッカーの確認）
  2. MCP `add_labels` を呼ぶ（`number`: open の subsystem Issue 番号・`is_pr`: false・`labels`: `$AI_MONITOR_LABEL_CONFIRM_SUBSYSTEM_CONDUCTOR` の値）

### ラベル除去

MCP `transition_phase` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false
- `remove_labels_`:
  - `$AI_MONITOR_LABEL_CONFIRM_STORY_CONDUCTOR` の値
- `add_labels_`: なし

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `story-conductor`
- `number`: $issue_number
