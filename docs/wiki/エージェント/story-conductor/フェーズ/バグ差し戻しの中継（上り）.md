# バグ差し戻しの中継（上り）

subsystem-conductor の修正完了報告（バグ差し戻し由来）を受けて、story Issue を再クローズし epic-conductor に遡上報告する。

## 手順

### 全子 subsystem の確認

初期処理で取得した Sub-issue の state で、全 subsystem が closed であることを確認する（未 close があれば何もせず自ターン終了）。

### 完了報告の Resolve

MCP `resolve_comments` で subsystem-conductor の修正完了報告コメントを Resolve する。

### story Issue の再クローズ

MCP `close` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false
- `reason`: `completed`

（修正は epic ブランチへ反映済み・story の成果物に変更なし）

### epic-conductor への遡上報告

MCP `add_labels` を呼ぶ:
- `number`: 親 epic Issue 番号
- `is_pr`: false
- `labels`:
  - `$AI_MONITOR_LABEL_CONFIRM_EPIC_CONDUCTOR` の値

続けて MCP `comment` を呼ぶ:
- `number`: 親 epic Issue 番号
- `is_pr`: false
- `sender`: `story-conductor`
- `receiver`: `epic-conductor`
- `body`: story のバグ修正完了報告（対象 story Issue 番号 + 修正内容の要約）

### ラベル除去

MCP `transition_phase` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false
- `remove_labels_`:
  - `$AI_MONITOR_LABEL_CONFIRM_STORY_CONDUCTOR` の値
- `add_labels_`: なし

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `story-conductor`
- `number`: $issue_number
