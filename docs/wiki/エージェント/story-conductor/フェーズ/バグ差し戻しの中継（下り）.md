# バグ差し戻しの中継（下り）

epic-conductor のバグ差し戻し（story reopen + バグ内容コメント）を受けて、該当 subsystem Issue を reopen して差し戻す。

## 手順

### 該当 subsystem の特定

バグ差し戻しコメントの fail 内容から、修正が必要な subsystem を特定する（story 本文の `## 実装分担` から番号を得る）。

### subsystem への差し戻し

該当する各 subsystem Issue について実施する:

1. MCP `reopen_issue` を呼ぶ（`number`: 該当 subsystem Issue 番号）
2. MCP `comment` を呼ぶ（`number`: 該当 subsystem Issue 番号・`is_pr`: false・`sender`: `story-conductor`・`receiver`: `subsystem-conductor`・`body`: fail 内容 + 修正方針）
3. MCP `add_labels` を呼ぶ（`number`: 該当 subsystem Issue 番号・`is_pr`: false・`labels`: `$AI_MONITOR_LABEL_CONFIRM_SUBSYSTEM_CONDUCTOR` の値）

### バグ差し戻しコメントへの返信と Resolve

MCP `reply_comment` を呼ぶ:
- `comment_node_id`: epic-conductor のバグ差し戻しコメントの node_id
- `sender`: `story-conductor`
- `receiver`: `epic-conductor`
- `body`: 中継結果（対象 subsystem Issue 番号）

続けて MCP `resolve_comments` で自分宛コメントを一括 Resolve する。

### ラベル除去

MCP `transition_phase` を呼ぶ:
- `number`: $issue_number
- `is_pr`: false
- `remove_labels_`:
  - `$AI_MONITOR_LABEL_CONFIRM_STORY_CONDUCTOR` の値
- `add_labels_`: なし（ユーザー承認なしの自動完了。story Issue は open のまま）

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `story-conductor`
- `number`: $issue_number
