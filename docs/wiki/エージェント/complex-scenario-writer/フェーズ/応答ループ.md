# 応答ループ

シナリオ作成後の待機中に来たユーザーフィードバックに応じてシナリオ md を修正 push し、再び待機に入る。

## 手順

### 内容に応じた対応

| フィードバックの種別 | 対応 |
| --- | --- |
| ノードの追加・削除・順序変更 | シナリオ md の mermaid 図を修正 |
| つなぎ目の粒度・focus 範囲の変更 | `subgraph FOCUS` の範囲やラベルを修正 |
| 異常シナリオの追加 | 該当シナリオファイルに `## 異常シナリオ（{条件}）` を追加 |
| 質問への回答 | 回答を該当コメントに返信追記（本文・シナリオファイルは触らない） |

シナリオ修正の場合は epic ブランチの worktree で:

- `git add docs/wiki/設計図/シナリオ/複合ユースケース/{機能名}.md`
- `git commit -m "{機能名}複合シナリオを{修正内容の要約}"`
- `git push`

修正内容は該当コメントに MCP `reply_comment` で返信追記する:
- `comment_node_id`: 返信対象コメントの node_id
- `sender`: `complex-scenario-writer`
- `receiver`: ユーザーログイン名
- `body`: 修正内容の要約

### assignee 再設定

MCP `set_assignee` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `complex-scenario-writer`
- `number`: $pr_number
