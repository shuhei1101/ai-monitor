# モックの応答ループ

モックの待機中に来たユーザーフィードバックに応じてモック HTML を修正 push し、再び待機に入る。

## 手順

### 内容に応じた対応

| フィードバックの種別 | 対応 |
| --- | --- |
| 特定画面のレイアウト・要素の修正 | 該当 HTML を修正して push |
| 遷移リンクの修正 | 該当 HTML の遷移リンクを修正して push |
| 共通コンポーネントの修正 | `docs/mock/components/` を修正して push（利用画面すべてに反映される） |
| 質問への回答 | 回答内容だけ返信追記（本文・HTML は触らない） |

修正内容は該当コメント（画面 URL コメント or ユーザーコメント）に MCP `reply_comment` で返信追記する:
- `comment_node_id`: 返信対象コメントの node_id
- `sender`: `mock-designer`
- `receiver`: ユーザーログイン名
- `body`: 修正内容の要約（URL は変わらないため再共有不要）

### assignee 再設定

MCP `set_assignee` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `mock-designer`
- `number`: $pr_number
