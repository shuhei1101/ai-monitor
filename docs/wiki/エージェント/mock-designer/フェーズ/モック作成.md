# モック作成

方針の合意（`議論中` 除去）を受けて、確定した画面一覧のモック HTML を `docs/mock/` 配下に作成 push し、PR 本文に `### モック` を追記して 1 画面 = 1 コメントで URL を共有して待機に入る。

## 手順

### 自分宛コメントの選別

未回答・未対応の自分宛コメントが残る場合は「方針の応答ループ」に戻ってユーザーに確認質問を投げる（以降の手順は実行しない）。

### 方針関連コメントの一括 Resolve

MCP `resolve_comments` を呼ぶ:
- `node_ids`: 方針の応答ループで処理した自分宛コメントの `node_id` 配列（epic-conductor の指示コメントとモック URL コメントは含めない・完了処理で Resolve）

### モック HTML の作成

`規約/モック画面構成.md` に従って、`### 画面一覧` の各画面について `docs/mock/pages/{画面名}/issues/{epic 番号}/{案名}/index.html` を作成する（PR ブランチに push）。

- 共通コンポーネントは `docs/mock/components/` を再利用
- スタイルは `docs/mock/styles/` を利用（`規約/デザインスタイル.md` 準拠）
- 遷移リンクは相対パスで他の画面 HTML へつなぐ

### PR 本文に `### モック` を追記

MCP `update_body` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true
- `body`: 既存本文（`### 画面一覧` / `### 画面遷移` 記入済み）+ `### モック`（各画面の URL 一覧表 = raw.githack.com の URL）

`### モック` の URL 書式:

```
https://raw.githack.com/${REPO_SLUG}/{ブランチ名}/docs/mock/pages/{画面名}/issues/{epic 番号}/{案名}/index.html
```

- `${REPO_SLUG}` = セッション開始時に load-constants.sh が export 済み（`owner/repo` 形式）
- `{ブランチ名}` = `git branch --show-current` の実値（worktree の現在ブランチ = epic ブランチ）
- `{画面名}` / `{epic 番号}` / `{案名}` は方針で確定した値

### 1 画面 = 1 コメントで URL 共有

各画面について MCP `comment` を呼ぶ（画面数分繰り返す）:
- `number`: $pr_number
- `is_pr`: true
- `sender`: `mock-designer`
- `receiver`: ユーザーログイン名
- `body`: 画面名 + 概要 + URL + レビュー観点（レイアウト・遷移・命名等）

### 議論中 付与 + 待機

MCP `add_labels` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true
- `labels`:
  - `$AI_MONITOR_LABEL_IN_DISCUSSION` の値

続けて MCP `set_assignee` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `mock-designer`
- `number`: $pr_number
