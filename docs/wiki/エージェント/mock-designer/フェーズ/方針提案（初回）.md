# 方針提案（初回）

epic-conductor の指示コメントと親 epic Issue を確認し、既存画面を調査したうえで、epic PR 本文の `## UI 設計` に `### 画面一覧` と `### 画面遷移` の草案を追記して待機に入る。

## 手順

### 指示コメントと親 epic Issue の確認

初期処理で取得した未解決の自分宛コメントから epic-conductor の指示コメント（画面方針の要点）を読む。

続けて親 epic Issue の内容を取得する。
PoC PR 本文の `## 紐づく Issue` から親 epic Issue 番号を得て MCP `get_issue_or_pr` を呼ぶ:
- `number`: 親 epic Issue 番号
- `is_pr`: false
- `comments`: false

epic Issue 本文の `## ユースケース一覧` と `## 横断要件` から画面の元ネタを抽出する。

### 既存画面・共通コンポーネントの調査

`docs/mock/pages/` と `docs/mock/components/` の索引を確認し、流用可能な既存画面・共通コンポーネントを把握する（`規約/モック画面構成.md` 参照）。

### PR 本文に方針草案を追記

テンプレート（PR本文/エピック）の `## UI 設計` に `### 画面一覧` と `### 画面遷移` の草案を追記する。

- `### 画面一覧`: 新規 / 変更の別 + 画面名 + 概要の表（`規約/モック画面構成.md` の粒度に従う）
- `### 画面遷移`: Mermaid の `flowchart` で全体遷移図

MCP `update_body` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true
- `body`: 既存本文 + `## UI 設計` + `### 画面一覧` + `### 画面遷移`

### 完了報告の投稿

MCP `comment` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true
- `sender`: `mock-designer`
- `receiver`: ユーザーログイン名（`gh api user --jq '.login'` で取得）
- `body`: 方針草案の完了報告（画面一覧の要点 + 遷移の主要導線 + 承認・修正方法の案内）

### 議論中 付与 + 待機

MCP `add_labels` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true
- `labels`:
  - `$AI_MONITOR_LABEL_IN_DISCUSSION` の値

続けて MCP `set_assignee` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `mock-designer`
- `number`: $pr_number
