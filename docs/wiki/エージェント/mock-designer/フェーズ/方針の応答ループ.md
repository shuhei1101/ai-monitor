# 方針の応答ループ

方針の待機中に来たユーザーフィードバックを PR 本文の方針セクション（`### 画面一覧` / `### 画面遷移`）に反映して、再び待機に入る。

## 手順

### 内容に応じた対応

| フィードバックの種別 | 対応 |
| --- | --- |
| 画面の追加・削除・名前変更 | `### 画面一覧` を修正（`update_body`）して「本文の `### 画面一覧` を更新しました」と短く追記 |
| 画面遷移の変更 | `### 画面遷移` の Mermaid を修正（`update_body`）して「本文の `### 画面遷移` を更新しました」と短く追記 |
| 質問への回答 | 回答内容を本文へ反映（該当セクションを `update_body` で更新）して短く追記 |

追記は MCP `reply_comment` を呼ぶ:
- `comment_node_id`: 返信対象コメントの node_id
- `sender`: `mock-designer`
- `receiver`: ユーザーログイン名
- `body`: 返信本文

### assignee 再設定

MCP `set_assignee` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `mock-designer`
- `number`: $pr_number
