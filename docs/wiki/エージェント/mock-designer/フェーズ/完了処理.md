# 完了処理

モックの合意（`議論中` 除去）を受けて、PR の `確認:mock-designer` を除去し、親 epic Issue に確認ラベルを渡して役割を終える。
PR 本文の `## UI 設計` はすでに 3 セクション（`### 画面一覧` / `### 画面遷移` / `### モック`）が段階的に記入済みなので、本手順では触らない。

## 手順

### 自分宛コメントの選別

未回答・未対応の自分宛コメントが残る場合は「モックの応答ループ」に戻ってユーザーに確認質問を投げる（以降の手順は実行しない）。

### 一括 Resolve

MCP `resolve_comments` を呼ぶ:
- `node_ids`: 自分宛コメントの `node_id` 配列（epic-conductor の指示コメント + モック URL コメントを含む全て）

### ラベル除去

MCP `transition_phase` を呼ぶ:
- `number`: $pr_number
- `is_pr`: true
- `remove_labels_`:
  - `$AI_MONITOR_LABEL_CONFIRM_MOCK_DESIGNER` の値
- `add_labels_`: なし

### 確認ラベルの引き継ぎ

$pr_number の PR 本文の `## 紐づく Issue` から親 epic Issue 番号を得る。

MCP `add_labels` を呼ぶ:
- `number`: 親 epic Issue 番号
- `is_pr`: false
- `labels`:
  - `$AI_MONITOR_LABEL_CONFIRM_EPIC_CONDUCTOR` の値

### 完了報告の投稿

MCP `comment` を呼ぶ:
- `number`: 親 epic Issue 番号
- `is_pr`: false
- `sender`: `mock-designer`
- `receiver`: `epic-conductor`
- `body`: UI 設計の完了報告（画面一覧のサマリ + モック URL の一覧 + PR 本文への記録済みの旨 + 確認後の Resolve 依頼）

### 作業完了報告

MCP `report_completion` を呼ぶ:
- `agent_name`: `mock-designer`
- `number`: $pr_number
